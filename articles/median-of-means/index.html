Median-of-Means Estimator

This article starts from a single extreme value, then turns the intuition into a notebook-style Monte Carlo experiment. The goal is not to claim that Median-of-Means is always best; the goal is to show how it reduces tail risk of estimation error under contaminated or heavy-tailed noise.

Date 2026-06-30 Format Notebook experiment Concept Robust estimation Output 代码 · 表格 · 图表

Before the notebook

What this article is trying to prove

普通均值不是“坏”的估计器；它的问题是对数值大小完全信任。当样本里出现少量极端值时，均值可能偶尔错得很离谱。 MoM（分组均值中位数）的价值是把这个风险拆成两层：组内仍然求均值，组间用中位数聚合。

01 一个异常值可以支配均值

单个 1000 会把样本均值从接近 10 的视觉中心拉到 109.2。

02 MoM 改变聚合问题

它问的是“分组均值里典型的均值是什么”，而不是一次性平均所有原始数值。

03 蒙特卡洛让判断可检验

重复实验可以比较普通均值、中位数和 MoM 的误差分布。

04 边界条件很重要

MoM 降低污染噪声下的尾部风险；它不是在所有干净场景里方差都更小的通用替代品。

步骤 02 / 估计器思路

先分组，再取中位数

MoM（分组均值中位数）的做法是先分组，每组算均值，最后对这些组均值取中位数。这样异常值会污染其中一组，但只要被污染的组不到一半，中位数聚合就能把它压下去。

核心问题变了： 普通均值问：“所有点的平均值是多少？” MoM 问：“许多小组里的典型平均值是多少？”

步骤 03 / 一般形式

把估计器写清楚

更一般地，把样本分成 \(k\) 个组 \(G_1,\ldots,G_k\)，先计算每组内部均值，再对这些组均值取中位数：

\[ \hat\mu_{\mathrm{MOM}} = \operatorname{median}\left(\bar X_1,\ldots,\bar X_k\right), \qquad \bar X_j = \frac{1}{|G_j|}\sum_{i\in G_j} X_i \]

这和重尾分布的直觉相连：当一个分布或抓取来的数据集可能产生极端值时，普通均值在理论上可能有稳定中心，但在有限样本里会出现很不稳定的实现。

笔记实验

重复污染过程

单个异常值例子只能说明“可能出问题”；重复随机实验才能看到普通均值的误差分布是否真的更不稳定。 每次生成 200 个样本，其中 95% 来自以 10 为中心的正常噪声，5% 来自仍然以 10 为中心、但波动极大的极端噪声。重复 5000 次后，比较普通均值、普通中位数和 MoM。

代码 · 运行模拟点击隐藏 
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

rng = np.random.default_rng(42)

def median_of_means(x, k=20):
    x = np.array(x, copy=True)
    rng.shuffle(x)
    blocks = np.array_split(x, k)
    block_means = [block.mean() for block in blocks]
    return np.median(block_means)

def generate_sample(n=200, true_center=10, contamination_rate=0.05):
    x = rng.normal(true_center, 1, size=n)
    mask = rng.random(n) < contamination_rate
    x[mask] = rng.normal(true_center, 50, size=mask.sum())
    return x

def run_experiment(repeats=5000, n=200, true_center=10):
    records = []
    for _ in range(repeats):
        x = generate_sample(n=n, true_center=true_center)
        records.append({
            "sample_mean": np.mean(x),
            "median": np.median(x),
            "median_of_means": median_of_means(x, k=20),
        })
    return pd.DataFrame(records)

results = run_experiment()

限制与实际使用

MoM 降低的是重尾噪声下的估计尾部风险，而不只是普通意义上的方差。它适合提醒我们：在薪资异常值、异常交易金额、长尾响应时间或带噪声的网页抓取记录中，先问均值稳定需要什么假设。

  • MoM 依赖组数 \(k\)；分组不佳或被污染的组过多时仍然会受影响。
  • 样本中位数在估计中心时可能看起来最好，但它和均值目标并不完全相同。
  • 这篇文章是推理演示，不是通用估计器推荐。

结论：有用的习惯是检查估计器的尾部风险，而不仅是平均误差。
